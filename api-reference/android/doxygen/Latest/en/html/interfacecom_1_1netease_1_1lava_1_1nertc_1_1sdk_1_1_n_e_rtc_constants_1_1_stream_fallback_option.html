◆ AUDIO_ONLY

int com.netease.lava.nertc.sdk.NERtcConstants.StreamFallbackOption.AUDIO_ONLY = RTCStreamFallbackOption.AUDIO_ONLY
  • When uplink networks are poor, publish audio streams only.
  • When downlink networks are poor, first try to receive low-quality video streams with low resolution and bitrate. If network conditions don't display videos, return to receive audio-only streams only.

◆ DISABLED

int com.netease.lava.nertc.sdk.NERtcConstants.StreamFallbackOption.DISABLED = RTCStreamFallbackOption.DISABLED

No fallback behavior for the locally published video stream when the uplink network condition is poor. The stream quality is not guaranteed.

Note
The option is only applicable to the setLocalPublishFallbackOption method but the setRemoteSubscribeFallbackOption method.

◆ VIDEO_STREAM_LOW

int com.netease.lava.nertc.sdk.NERtcConstants.StreamFallbackOption.VIDEO_STREAM_LOW = RTCStreamFallbackOption.VIDEO_STREAM_LOW

Under poor downlink network conditions, the SDK will only receive low-quality video streams with low resolution and low bitrate.

Note
The option is only applicable to setRemoteSubscribeFallbackOption method but setRemoteSubscribeFallbackOption method.
