NERtcAudioFrame  audioFrame

Gets the audio data from specific remote users before the audio stream is mixed with other audio streams.
After registering the audio viewer, if remote users subscribe remote audio (subscription by default) and enable it, the SDK triggers the callback to send you the audio data when the system captures video data before the mix.

Note
The audio data that is returned has only read permissions.
Deprecated:
This API is about to be deprecated. Use NERtcAudioFrameObserver#onPlaybackAudioFrameBeforeMixingWithUserID(long, NERtcAudioFrame, long) instead. In multi-channel scenarios, channelId is used to identify different channels.
Parameters
userIDRemote users ID.
audioFrameFor information about PCM video frame data, see audio.NERtcAudioFrame.

Gets the audio substream data from a specified remote user before mixing audio.
After the audio observer is registered, the SDK will get the audio data from a specified remote user before mixing if the remote audio substream is subscribed to and the remote user publishes the audio substream.

Note
The returned audio data can only be read.
Parameters
userIDA remote user ID.
audioFramePCM audio frame data. For more information, see audio.NERtcAudioFrame.

Returns the data of the local audio substream, used for the custom audio substream.

Note
  • If the substream is enabled, the SDK will return the data.
